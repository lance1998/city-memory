## 2025-07-07 - Global Keyboard Accessibility for Pseudo-Buttons
**Learning:** Adding `tabindex="0"` and `role="button"` makes `div`/`span` elements focusable and semantic, but it does *not* automatically trigger `click` handlers when pressing `Enter` or `Space` (unlike native `<button>` elements). This is a common accessibility trap in apps that heavily rely on pseudo-buttons.
**Action:** When adding accessibility to non-native interactive elements across an application, establish a global `keydown` event listener to delegate `Enter` and `Space` key presses to `document.activeElement.click()`, ensuring consistent keyboard accessibility without having to refactor every single component.

## 2025-07-18 - Dynamic ARIA Labels for Icon-Only Metric Buttons
**Learning:** For interactive UI elements (like "like" or "comment" buttons) that visually display only an icon and a numeric count (e.g., `<i class="fa-heart"></i><span>15</span>`), screen readers will read only the number "15", providing zero context. This is highly confusing. Furthermore, if these elements are generated dynamically via JavaScript (or within modals), patching them requires mutation observers.
**Action:** When dynamically adding accessibility attributes to such pseudo-buttons inside observers (like `MutationObserver`), check if the inner text is purely numeric. If so, inspect the child icon classes (e.g., `.fa-heart`) and the active state to dynamically construct and inject a descriptive compound `aria-label` (e.g., "取消点赞, 15"). This ensures blind users receive both context and metric value.

## 2025-07-22 - Dynamic Accessibility Patching for Reused Fragments
**Learning:** In applications where UI elements like detail modals are heavily reused and populated by disparate rendering fragments (e.g., `app-part1.js`, `app-part2.js`), attempting to patch ARIA attributes at the source HTML strings can be brittle or scattered. The application uses a central `#detail-content` container whose contents are dynamically replaced via `innerHTML`.
**Action:** Utilize a robust `MutationObserver` on the static parent container (`#detail-content`) to retroactively inject `role="button"`, `tabindex`, and dynamically computed compound `aria-label`s based on the newly inserted elements' state classes (e.g., `.active`) and inner text content. This safely isolates accessibility improvements without risking HTML generation string concatenation bugs across multiple fragment files.

## 2025-07-22 - Graceful Fallback for Icon-Only Buttons
**Learning:** When attempting to extract dynamic text from sibling elements (like `span` inside a button) to construct an `aria-label`, strictly requiring the presence of that sibling (e.g., `if (icon && span)`) will inadvertently break accessibility for buttons that consist entirely of an icon (e.g., `<button><i></i></button>`).
**Action:** Relax the requirement to just the primary identifier (the icon itself). Retrieve sibling text if it exists (`var countText = span ? span.textContent.trim() : '';`), but construct and apply the base `aria-label` regardless of the sibling's existence to ensure all variants are covered.

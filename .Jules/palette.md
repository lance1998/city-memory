## 2025-07-07 - Global Keyboard Accessibility for Pseudo-Buttons
**Learning:** Adding `tabindex="0"` and `role="button"` makes `div`/`span` elements focusable and semantic, but it does *not* automatically trigger `click` handlers when pressing `Enter` or `Space` (unlike native `<button>` elements). This is a common accessibility trap in apps that heavily rely on pseudo-buttons.
**Action:** When adding accessibility to non-native interactive elements across an application, establish a global `keydown` event listener to delegate `Enter` and `Space` key presses to `document.activeElement.click()`, ensuring consistent keyboard accessibility without having to refactor every single component.
## 2025-07-19 - Accessible Dynamic Action Pseudo-Buttons in Modal
**Learning:** For interactive pseudo-buttons (`.detail-action`) dynamically injected into modals (`#detail-content`) that visually display an icon and a number (e.g., likes or comments count), screen readers fail to convey the action's intent or state. Furthermore, when the UI relies on MutationObservers (like `contentObserver`) to patch accessibility, we must construct and dynamically update a compound `aria-label` (e.g., "取消点赞, 15") based on both the button's internal text and state classes to provide adequate context.
**Action:** When adding accessibility to icon-text buttons displaying counts in dynamic containers, use a MutationObserver to apply `role="button"` and `tabindex="0"`, and always construct a compound `aria-label` merging the action's state (e.g., "点赞" vs "取消点赞") and the count.
